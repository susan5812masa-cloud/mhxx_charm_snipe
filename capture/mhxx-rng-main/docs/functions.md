各スキル名の並びは、ID順になっています。必要なら、Kiranicoなどを参照してください。


### `set_blue()`

風化したお守りを検索するモードに設定します。

`set_red()`なら古びたお守り、`set_yellow()`なら光るお守りを検索するモードに設定されます。

光るお守りは充足値によって最大スロット数が異なるので注意してください。


### ■

```python
p = parameter('第1スキル',第1SP,'第2スキル',第2SP,スロット数,'原産地')

for num1 in range(◯):
  search(*p)
```

指定されたお守りを◯個先の乱数まで検索し、乱数位置を表示します。

スキル名は3rdのセルに書かれているものを入力してください。スキル名が正しく入力されないとエラーになります。

'原産地'には、マカ錬金なら'マカ'、クエストなら'炭鉱'を入力してください。他言語の場合は、3rdのセルの`origin`の要素を入力してください。

1000万回のループなら15秒程で終わります。

下に時間が表示されますが参考程度に留めておいてください。

`search`の代わりに`search_greater`を使うと、指定された第1スキル、第2スキルを持ち、指定された値以上の第1SP、第2SP、スロット数を持つお守りを◯個先の乱数まで検索し、乱数位置とお守りを表示します。

下の「● aim points exist」や◯、Xといった表示は、当たりフレームの有無を表しています。右端の[◯]や[X]は検索中のお守りの位置を表します。

`search_greater_skill1`を使うと、指定された第1スキルを持ち、指定された値以上の第1SP、スロット数を持つお守りを◯個先の乱数まで検索し、乱数位置とお守りを表示します。

`search_greater_skill2`を使うと、指定された第2スキルを持ち、指定された値以上の第2SP、スロット数を持つお守りを◯個先の乱数まで検索し、乱数位置とお守りを表示します。


### ■

for i in range(◯):

### ■

```python
p = parameter('第1スキル',第1SP,'第2スキル',第2SP,スロット数,'原産地')

show_fast(◯,'関数名',p)
```

指定された条件を満たすお守りを、◯個先の乱数まで検索します。

search系統の200倍の速度で検索可能です。初回のみ若干起動が遅いです。

大きすぎる値を入力すると停止しないかもしれません。

'関数名'には、'search'、'search_greater'、'search_greater_skill1'、'search_greater_skill2'のいずれかを入力して条件を指定してください。


### `show_fast_multi(◯,お守り1,お守り2,お守り3,…)`

いずれかのお守り以上の条件を持つお守りを、◯個先の乱数まで検索します。

後ろの部分には、`parameter()`で設定した、1個以上の好きな数のお守りを入力できます。

マカで検索するかクエストで検索するかは、最初に引数として入力したお守りで決まります。下の例では、p1が'マカ'なので、他のお守りも'マカ'で検索されます。

`show_fast()`より少し遅く、お守りの数が増えると更に少し遅くなります。

**使用例**

```python
p1 = parameter('斬味',5,'痛撃',5,3,'マカ')
p2 = parameter('斬味',5,'会心',5,3,'マカ')
p3 = parameter('痛撃',6,'匠　',4,3,'マカ')
p4 = parameter('痛撃',6,'斬味',5,2,'マカ')

show_fast_multi(10**7,p1,p2,p3,p4)
```


### `around(◯,□,'原産地')`

乱数位置◯の前後□個のお守りを全て表示します。

◯には`search()`等で求めたお守りの乱数位置を入力してください。


### `jujupush(◯,□)`

乱数位置◯、ランク□でマカフシギ錬金をしたときの結果を表示します。

□を入力しなければ、最高ランクの結果が表示されます。


### `halcyonpush(◯,□)`

乱数位置◯、ランク□で天運の錬金をしたときの結果を表示します。

□を入力しなければ、最高ランクの結果が表示されます。


### `sequence_charm(◯,□)`

乱数位置◯から、クエスト報酬で□個同時にお守りを鑑定したときの結果を表示します。


### ■

```python
reward_table = {報酬テーブル}
reward = '獲得した報酬'

search_reward(初期乱数位置, 検索数, 運気スキルによる閾値, reward, reward_table)
```

採取ツアーで得たクエスト報酬から、クエスト報酬を生成する直前の乱数位置を割り出し表示します。

報酬生成時には乱数が消費され、例えば採取ツアーでは5〜12個の乱数が、報酬数に応じて追加で消費されていることに注意してください。（報酬数が4,5,6,7,8なら、消費量は5,7,9,11,12）

報酬が生成される瞬間にタイマーをスタートすれば、`search_reward`で出力されたフレームに、そのタイマーの経過時間に相当するフレームと、報酬生成による乱数消費量を足したものが、ゲームで現在進行中の乱数位置となります。

`reward_table`には、そのクエストの報酬テーブルを記入してください。

`reward`には、そのクエストで入手した報酬の1段目、最大8個のアイテム名を、空白区切りの文字列で入力してください。

運気スキルによる閾値には、運気スキルを発動していない場合は22、幸運を発動している場合は25、強運を発動している場合は28、激運を発動している場合は31を入力してください。

`reward_table`を適切に書き換えれば、他のクエストのクエスト報酬からでも乱数位置を特定できると思います。

`reward_table`に記入するアイテム名は、利用者が識別できるなら何でも良いですが、`reward_table`と`reward`で同じ名称を使ってください。

検索数が1億なら1秒ほどで終わります。初回のみ若干起動が遅いです。

**使用例**

```python
reward_table = {'謎骨':20,'釣り':40,'生肉':65,'砥石':85,'力餌':90,'重餌':95,'速餌':100}
reward = '力餌　釣り　生肉　謎骨　謎骨　生肉　釣り'

search_reward(0, 10**7, 28, reward, reward_table)
```


### ■

```python
combo = '調合数の記録'
search_combo(初期乱数位置, 検索数, combo)
```

調合による弾所持数の変化から、調合を開始した際の乱数位置を割り出し、そのフレームに調合による乱数進行（調合回数×2）を足したものを表示します。

調合開始と同時にタイマーをスタートすれば、`search_combo`で出力されたフレームに、そのタイマーの経過時間に相当するフレームを足したものが、ゲームで現在進行中の乱数位置となります。

'調合数の記録'にはココット村の自宅で、アイルーのルームサービスやオトモがいない状態で、Lv2通常弾を連続調合した際の所持数の変遷を、空白で区切り入力してください。

必ずしも0から始め99で終わる必要はありませんが、調合数の記録は最初から最後まで入力してください。

合計60個ほど調合すれば、十分特定できると思います。

乱数の不定消費により適切に検索できない場合は、調合数の記録を短くしたり、再度調合した際の記録を入力してください。

検索数が1億なら1秒ほどで終わります。初回のみ若干起動が遅いです。

**使用例**

```python
combo = '''
00 03 06 09 11 13 16 18 21 23
25 27 30 32 35 38 41 44 47 50
52 55 58 61 65 69 72 75 78 81
85 88 91 93 97 99'''

search_combo(0, 10**7, combo)
```

### `num_of_charms`

### `showallrank`

マカ錬金で、古びたお守りや光るお守りを検索した際、これがTrueなら、全てのランクでの当たりフレームを表示します。Falseなら、最高ランクのみの当たりフレームを表示します。


### `showrare`

Trueならお守りのレア度を表示、Falseなら非表示にします。
